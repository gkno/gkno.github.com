faq
  • I run the same pipeline many times and have to change one of the parameters every time. Can I modify the pipeline, so that it uses default values that are more useful to my work?

    Yes. You can create a modified configuration file using your own parameters. See the Modifying an existing configuration file tutorial for details.
  • Can I force output files to contain pipeline parameters? I run a pipeline on the same input data, but with different parameters. I want to make sure that the output files have different names without having to set names on the command line.

    Yes, you can add a section to the pipeline configuration file describing how you want to construct the output filenames. See the documentation or the Constructing output filenames tutorial for details of how to use this functionality.
  • Can I run the same pipeline multiple times, for different input files?

    Yes, gkno is constructed so that a list of inputs (and other parameters, if desired) can be provided in a json file and a makefile will be generated for each set of inputs and executed (if desired). See the Performing multiple runs of a pipeline. Note that this feature can be used to run the same pipeline multiple times with the same input files, but different parameters.
