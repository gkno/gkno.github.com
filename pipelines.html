Pipelines
Pipes are a way to string together several tools so that the output of one tool flows into the input of another. gkno comes prepackaged with the several pipelines listed here. Also, custom pipelines can be built by creating a new config file.

An easy way to understand what a pipeline does or how to alter one, is to look at the pipeline maps. To create the maps, we took inspiration from our own subway system here in Boston ( the T ). Just as the T map helps you get from point A to point B, the pipeline maps help you take what you have (e.g a fastq file) and get to where you want to go (e.g. a vcf) while clearly illustrating all the stops along the way. Though we only have the red (pipe)line running at the moment, more lines will come online in the future.
